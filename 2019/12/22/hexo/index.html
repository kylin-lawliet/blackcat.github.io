    本篇文章主要是整理记录使用Hexo建站过程笔记以及踩过的坑

    安装

    Win安装node，npm

    • 下载node:node下载地址

    • 解压node:创建两个文件node-cache，node-global用来指定npm的模块路径和缓存路径

    • 配置node:在系统变量下新建一个NODE_HOME变量，把新建的NODE_HOME添加到PATH中
    NODE_HOME
D:\Software\node-v12.14.0-win-x64\

    %NODE_HOME%
%NODE_HOME%\node-global

    • 下载npm:打开cmd，进入node解压文件夹下，配置刚才新建的两个文件夹
    npm config set prefix "D:\Software\node-v12.14.0-win-x64\node-global"
npm config set cache "D:\Software\node-v12.14.0-win-x64\node-cache"
    • 配置npm:设置淘宝为国内镜像源(相当于maven设置阿里是国内的远程镜像仓库)
    npm config set registry https://registry.npm.taobao.org
    • 其他命令
    npm config get registry // 查看设置的国内镜像对不对
node --version // 查看安装的node版本
npm --version // 查看安装的npm版本

    Win安装 hexo

    创建一个文件夹，存放hexo文件

    cd 到创建的文件夹下
npm install -g hexo-cli // 下载hexo
hexo init // 初始化这个文件夹
hexo generate // 生成默认主题和默认文章-Helloworld
hexo server //启动服务

    http://localhost:4000/ 本地预览

    文件夹目录

    • node_modules: 依赖包
    • public：存放生成的页面
    • scaffolds：生成文章的一些模板
    • source：用来存放你的文章
    • themes：主题文件
    • ** _config.yml: 博客的配置文件**

    修改主题

    主题下载地址hexo.io/themes，wiki/Themes，zhihu

    • 打开git bath
    cd 到hexo安装文件夹根目录
git clone https://github.com/MOxFIVE/hexo-theme-yelee.git themes/yelee

    修改hexo根目录下的_config.yml 属性:theme: yelee

    更新主题

    • 如果作者有改动主题
    cd hexo安装文件夹根目录/themes/yelee
git pull origin master

    绑定个人域名

    • 1.购买域名(具体运营商，购买步骤自行百度)
    • 2.在域名控制台中>>看到购买的域名>>点解析进去
    • 3.添加两条CNAME解析(百度其他教程有说配置ip，但是不建议)
    • 4.登陆github选择gitpage仓库，选择settings，下滑找到Github Pages 在Custom domain里填入你的域名，然后点击Save或者在 hexo 项目下，source 文件夹下面创建 CNAME 文件（没有后缀名的），在里面写上你的域名。
    • 5.访问成功

    PS:
    如果想要本地访问域名使用
    把记录类型改为:A，记录值改为:本地ip，但是与上方配置不能同时存在
    绑定域名分两种情况:带www和不带www

    • 如果你填写的是没有www的，比如 mygit.me，那么无论是访问 http://www.mygit.me 还是 http://mygit.me，都会自动跳转到 http://mygit.me
    • 如果你填写的是带www的，比如 www.mygit.me，那么无论是访问 http://www.mygit.me 还是 http://mygit.me，都会自动跳转到 http://www.mygit.me
    • 如果你填写的是其它子域名，比如 abc.mygit.me，那么访问 http://abc.mygit.me 没问题，但是访问 http://mygit.me，不会自动跳转到 http://abc.mygit.me
      绑定了新域名之后 原来的xxx.github.io并没有失效，会自动跳转到你的新域名。

    备份Hexo博客源文件

    第一次备份

    在Github下创建一个新的仓库，取名为XXX(与本地的hexo源码文件夹同名)

    cd 本地的hexo文件夹
git init  加载仓库
git remote add origin git@github.com:kylin-lawliet/blackcat.git 设置远程仓库地址
git pull origin master 更新
git add . 添加到本地仓库
git commit -m "备注"  源码在本地的提交
git push origin master 将本地的仓库文件推送到Github

    更换开发环境

    克隆过来的文件没有之前下载的themes主题文件，直接把之前的文件复制过来，或再去Git下载主题。 

    git clone git@github.com:kylin-lawliet/blackcat.git 设置远程仓库地址
cd  blackcat (克隆后会创建一个文件夹，进入文件夹内)
修改文件后（或直接把之前环境文件复制过来）
git add . 添加到本地仓库
git commit -m "备注"  源码在本地的提交
git push origin master 将本地的仓库文件推送到Github

    更新同步

    git add .
git commit -m "源文件更新"
git push origin master

    远程仓库有更新

    git pull origin master

    还有另一种方式的shelljs模块因为懒没有去尝试

    踩过的坑

    记录的问题都是本人建站时所遇到的，仅供参考

    无法访问此网站

    别人的电脑都能访问你的网址，但你自己却访问不了。出现此问题，可能会是dns缓存的问题！

    ipconfig /flushdns

    出现报错

    • 1.找不到路径(菜单连接配置路径不正确)
    • 2.修改config.yml没有重新启动服务
    • 3.页面(不是文章)中添加不正确属性 如:页面加入categories属性

    未找到相关的Issues进行评论

    • 先建一个名gitalk-commnet(名称随意)的评论仓库
    • 登陆github 进入seting 在左菜单找到Developer setting 进入选择OAuth Apps 新建

    没有域名就填写https://xxx.github.io/
    注册成功后可以看到client_id与client_secret

    修改主题配置文件_config.yml

    代码块样式不起作用

    将主题配置文件中highlight关闭然后运行 在修改回来修改样式之后执行命令

    hexo clean & hexo g & hexo s

    hexo命令

    创建命令

    hexo new “postName” 新建文章
    hexo new page “pageName” 新建页面
    hexo new page tags 创建标签页面
    hexo new page categories 创建分类页面
    hexo new page about 创建关于我页面

    常规命令

    hexo generate 生成静态页面至public目录
    hexo server 开启预览访问端口（默认端口4000，ctrl + c关闭server）
    hexo deploy 部署到GitHub
    hexo help 查看帮助
    hexo version 查看Hexo的版本

    简化命令

    hexo g == hexo generate
    hexo d == hexo deploy
    hexo s == hexo server
    hexo s -g -p 4001 hexo指定端口运行
    hexo s -g 生成并本地预览
    hexo d -g 生成并上传
